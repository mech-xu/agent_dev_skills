# DevFlow Automation

AI-optimized project development automation toolkit for AI coding agents.

## Features

- **Agent Execution Log** - Structured log format optimized for AI agent reading and execution
- **Log Merge** - Intelligently merge development logs into daily or target-date logs
- **Auto Deploy** - One-command deployment for Worker + Mini Program projects
- **Context Generation** - Integrated with repowise and llm-context for AI context optimization

## Installation

```bash
npm install -g devflow-automation
```

### Peer Dependencies

```bash
npm install -g repowise llm-context
```

## Tools

### 1. Log Merge (`devflow-log-merge`)

Intelligently merge development logs in Obsidian vault.

```bash
# List all log files
devflow-log-merge list

# Merge logs to today
devflow-log-merge merge

# Merge logs to specific date
devflow-log-merge merge 2026-04-10

# View today's log
devflow-log-merge today
```

**Environment Variable:**
```bash
export OBSIDIAN_VAULT=/path/to/your/obsidian/vault
```

### 2. Auto Deploy (`devflow-deploy`)

One-command deployment for Cloudflare Worker + Mini Program projects.

```bash
devflow-deploy
```

Customize the script for your project structure.

### 3. Agent Execution Log Format

Structured format optimized for AI agent efficiency:

```markdown
# Project Agent Execution Log

## EXEC_SUMMARY
- Project status and session summary

## DELTA_CHANGES
- [NEW/FIX/OPT/DEL] file - description

## API_INVENTORY
- HTTP method, endpoint, description

## DB_SCHEMA
- Table: columns

## DEPLOY_CMD
- Deployment commands

## NEXT_ACTIONS
- Pending items with priority

## CONTEXT_OPT
- Context optimization strategy
```

## Configuration

### Shell Aliases

Add to your `~/.zshrc` or `~/.bashrc`:

```bash
# DevFlow aliases
alias df-deploy='devflow-deploy'
alias df-context='llm-context . js -i node_modules -i .git'
alias df-log='devflow-log-merge'
alias df-ctx='repowise create'
```

### Environment Variables

```bash
# Obsidian vault path
export OBSIDIAN_VAULT=/path/to/vault

# Project root
export PROJECT_ROOT=/path/to/project
```

## How It Works

### Context Optimization Pipeline

```
Code Changes → Agent Log → Obsidian → AI Context
     ↓              ↓           ↓          ↓
  repowise    log-merge    knowledge   llm-context
```

1. **Code Changes** tracked by `repowise create`
2. **Agent Log** structured for efficient AI parsing
3. **Obsidian** serves as knowledge base
4. **AI Context** generated by `llm-context`

### Agent Reading Priority

1. `Agent_Execution_Log.md` - Current state (highest priority)
2. `Dev_Log.md` - Historical records
3. `llm-context.*.txt` - Code context
